Visión general

Esta guía explica cómo publicar ControlBox para que cualquier persona instale la plataforma en su propio VPS con un solo comando curl.

Dos servidores distintos No confunda el VPS de distribución (donde usted publica el instalador) con el VPS de cada cliente (donde se ejecuta la instalación completa de ControlBox).

Su VPS de distribución

Sirve archivos estáticos en install.grodtech.com: el bootstrap install.sh y el paquete .tar.gz. Puede ser un VPS pequeño (1 GB RAM).

VPS del usuario final

Ejecuta curl ... | bash, descarga su paquete, instala Docker, despliega PostgreSQL, API, panel, etc. Mínimo 4 GB RAM y 40 GB disco.

Para que el comando remoto funcione debe tener listo:

  1. Imágenes Docker publicadas en un registro (ghcr.io/grodtech/controlbox-api y controlbox-panel)
  2. Paquete del instalador generado con package.sh
  3. Archivos accesibles por HTTPS en su dominio de instalación

Arquitectura de distribución

Usuario final (VPS cliente) | | curl -fsSL https://install.grodtech.com/install.sh | bash v Su VPS CDN (install.grodtech.com) |-- install.sh (bootstrap, ~5 KB) |-- controlbox-installer-1.0.0.tar.gz (instalador completo) | v Instalador en VPS del cliente |-- Descarga imágenes: ghcr.io/grodtech/controlbox-api:1.0.0 |-- Descarga imágenes: ghcr.io/grodtech/controlbox-panel:1.0.0 |-- Despliega stack Docker completo |-- Crea primer tenant automáticamente v Panel listo: IP:PUERTO/ControlBox_Panel

Coincidencia de versión El nombre del .tar.gz debe ser exactamente controlbox-installer-{CONTROLBOX_VERSION}.tar.gz. Esa versión también se usa para etiquetar las imágenes Docker que el instalador descargará en el VPS del cliente.

Paso 1 — Publicar imágenes Docker en GHCR

El instalador de producción no compila el código en el VPS del cliente: descarga imágenes preconstruidas.

1.1 Crear token de GitHub Container Registry

  1. En GitHub: Settings → Developer settings → Personal access tokens
  2. Crear token con permiso write:packages y read:packages
  3. Iniciar sesión en el registro: 
    echo "TU_GITHUB_TOKEN" | docker login ghcr.io -u TU_USUARIO --password-stdin

1.2 Construir y subir API

Desde la raíz del repositorio en su máquina de desarrollo o CI:

export VERSION=1.0.0
export REGISTRY=ghcr.io/grodtech

cd backend
docker build -t ${REGISTRY}/controlbox-api:${VERSION} .
docker tag ${REGISTRY}/controlbox-api:${VERSION} ${REGISTRY}/controlbox-api:latest
docker push ${REGISTRY}/controlbox-api:${VERSION}
docker push ${REGISTRY}/controlbox-api:latest

1.3 Construir y subir Panel

El panel de producción usa la ruta base /ControlBox_Panel:

cd frontend
docker build \
  --build-arg NEXT_PUBLIC_BASE_PATH=/ControlBox_Panel \
  --build-arg API_PROXY_URL=http://api:8000 \
  -t ${REGISTRY}/controlbox-panel:${VERSION} .
docker tag ${REGISTRY}/controlbox-panel:${VERSION} ${REGISTRY}/controlbox-panel:latest
docker push ${REGISTRY}/controlbox-panel:${VERSION}
docker push ${REGISTRY}/controlbox-panel:latest

1.4 Hacer las imágenes públicas (si aplica)

En GitHub → Packages → cada imagen → Package settings → Change visibility → Public. Si permanecen privadas, los VPS de sus usuarios necesitarán autenticarse contra GHCR antes de instalar.

Registro propio Si no usa ghcr.io/grodtech, edite installer/templates/docker-compose.platform.yml y cambie las referencias de imagen antes de empaquetar el instalador.

Paso 2 — Empaquetar el instalador

En su máquina local, con el repositorio ControlBox clonado:

2.1 Revisar versión y URL

Edite installer/config/defaults.conf si necesita cambiar dominio o versión:

CONTROLBOX_VERSION=1.0.0
CONTROLBOX_INSTALL_URL=https://install.grodtech.com

2.2 Generar el paquete

cd installer
chmod +x package.sh
CONTROLBOX_VERSION=1.0.0 CONTROLBOX_INSTALL_URL=https://install.grodtech.com bash package.sh

Salida esperada:

Paquete creado: installer/dist/controlbox-installer-1.0.0.tar.gz
Bootstrap:      installer/dist/install.sh

2.3 Comprobar contenido del paquete

tar tzf dist/controlbox-installer-1.0.0.tar.gz | head -20

Debe incluir install.sh, lib/, templates/ y config/.

Paso 3 — Preparar el VPS de distribución

VPS ligero recomendado: Ubuntu 22.04/24.04, 1 vCPU, 1 GB RAM, 10 GB disco.

3.1 Requisitos

  • Acceso SSH como root o sudo
  • Dominio apuntando al VPS (ej. install.grodtech.com)
  • Puertos 80 y 443 abiertos

3.2 Instalar Nginx

sudo apt update
sudo apt install -y nginx certbot python3-certbot-nginx

3.3 Crear directorio web

sudo mkdir -p /var/www/install.grodtech.com
sudo chown -R $USER:$USER /var/www/install.grodtech.com
sudo chmod 755 /var/www/install.grodtech.com

Paso 4 — Subir archivos al VPS

Desde su PC (donde generó el paquete), use scp o rsync:

Opción A — SCP

cd installer/dist

scp install.sh root@IP_DE_SU_VPS:/var/www/install.grodtech.com/
scp controlbox-installer-1.0.0.tar.gz root@IP_DE_SU_VPS:/var/www/install.grodtech.com/

Opción B — RSYNC (recomendado para actualizaciones)

rsync -avz --progress installer/dist/ \
  root@IP_DE_SU_VPS:/var/www/install.grodtech.com/

Permisos en el servidor

ssh root@IP_DE_SU_VPS

cd /var/www/install.grodtech.com
chmod 644 install.sh controlbox-installer-1.0.0.tar.gz
chown -R www-data:www-data /var/www/install.grodtech.com
Tip Puede automatizar esto con GitHub Actions: al crear un tag v1.0.0, construir imágenes, empaquetar el instalador y subir por SSH/rsync al VPS de distribución.

Paso 5 — Configurar Nginx y SSL

5.1 Virtual host

sudo nano /etc/nginx/sites-available/install.grodtech.com
server {
    listen 80;
    listen [::]:80;
    server_name install.grodtech.com;

    root /var/www/install.grodtech.com;
    index index.html;

    location / {
        autoindex off;
        try_files $uri $uri/ =404;
    }

    location ~ \.(sh|tar\.gz)$ {
        add_header Cache-Control "public, max-age=300";
        add_header Access-Control-Allow-Origin "*";
        default_type application/octet-stream;
    }
}
sudo ln -sf /etc/nginx/sites-available/install.grodtech.com /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx

5.2 Certificado Let's Encrypt

sudo certbot --nginx -d install.grodtech.com

Paso 6 — Configurar DNS

TipoNombreValor
Ainstall (o install.grodtech.com)IP pública de su VPS de distribución

Espere la propagación DNS (minutos u horas) antes de probar HTTPS.

Paso 7 — Verificar publicación

Ejecute estas pruebas desde cualquier máquina con Internet:

7.1 Comprobar archivos

curl -fsSL -I https://install.grodtech.com/install.sh
curl -fsSL -I https://install.grodtech.com/controlbox-installer-1.0.0.tar.gz

Debe responder HTTP/2 200 (o 301 → 200).

7.2 Probar bootstrap sin instalar

curl -fsSL https://install.grodtech.com/install.sh | head -20

Debe mostrar el script bash de ControlBox con CONTROLBOX_INSTALL_URL.

7.3 Instalación de prueba en un VPS limpio

Use un VPS de prueba (no su CDN) con Ubuntu 22.04 y ejecute el comando interactivo o no interactivo de la sección siguiente.

No pruebe en el VPS de distribución El VPS que sirve install.sh solo aloja archivos estáticos. La instalación completa de ControlBox debe hacerse en un servidor dedicado distinto.

Comandos que usarán sus usuarios

Una vez publicado, sus clientes instalan ControlBox en su propio VPS con:

Instalación interactiva (recomendada)

El asistente pregunta dominio, tenant, contraseña y puerto del panel:

curl -fsSL https://install.grodtech.com/install.sh | bash

Instalación no interactiva (automatizada)

curl -fsSL https://install.grodtech.com/install.sh | \
  CONTROLBOX_PRIMARY_DOMAIN=tudominio.com \
  CONTROLBOX_ADMIN_EMAIL=admin@tudominio.com \
  CONTROLBOX_TENANT_NAME="Mi Empresa" \
  CONTROLBOX_TENANT_SLUG=mi-empresa \
  CONTROLBOX_TENANT_ADMIN_EMAIL=admin@mi-empresa.com \
  CONTROLBOX_TENANT_ADMIN_PASSWORD='ContraseñaSegura123!' \
  CONTROLBOX_TENANT_ADMIN_FULL_NAME="Administrador" \
  CONTROLBOX_PANEL_PORT=8475 \
  CONTROLBOX_ASSUME_YES=true \
  bash

Resultado esperado al finalizar

URL PANEL:     75.148.75.12:8475/ControlBox_Panel
USUARIO:       admin@mi-empresa.com
CONTRASEÑA:    ContraseñaSegura123!

El usuario luego completa en el panel: Settings → Producción (rotación de secretos, TOTP, umbrales de alertas).

Publicar una nueva versión

  1. Actualice CONTROLBOX_VERSION en installer/config/defaults.conf
  2. Construya y suba nuevas imágenes Docker con el tag de versión (ej. 1.0.1)
  3. Ejecute package.sh con la nueva versión
  4. Suba al CDN:
    • install.sh (siempre sobrescribir)
    • controlbox-installer-NUEVA_VERSION.tar.gz (archivo nuevo)
  5. Mantenga el .tar.gz de la versión anterior un tiempo por si hay instalaciones en curso
  6. Pruebe: curl -fsSL https://install.grodtech.com/install.sh | bash en un VPS de staging

Solución de problemas

curl: (22) The requested URL returned error: 404

  • Verifique que install.sh y el .tar.gz existen en /var/www/install.grodtech.com/
  • El nombre del tar debe coincidir exactamente con CONTROLBOX_VERSION embebida en install.sh
  • Revise sudo nginx -t y los logs en /var/log/nginx/error.log

No se pudo descargar el instalador

  • Confirme HTTPS y certificado válido en el dominio
  • Pruebe manualmente: curl -fsSL -O https://install.grodtech.com/controlbox-installer-1.0.0.tar.gz
  • Revise que CONTROLBOX_INSTALL_URL en defaults.conf sea el dominio correcto antes de empaquetar

Error al descargar imágenes Docker en el VPS del cliente

  • Imagen no publicada o tag incorrecto en GHCR
  • Paquete GHCR privado sin autenticación en el VPS del cliente
  • Versión en platform.env / defaults.conf no coincide con el tag subido

Cambiar de dominio (no usar grodtech.com)

  1. Edite CONTROLBOX_INSTALL_URL=https://install.sudominio.com en defaults.conf
  2. Regenere el paquete con package.sh
  3. Configure DNS, Nginx y SSL en su nuevo dominio
  4. Distribuya el nuevo comando: curl -fsSL https://install.sudominio.com/install.sh | bash
